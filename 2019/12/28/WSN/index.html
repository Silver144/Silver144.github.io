Routing

SPIN(negotiation-based, data-centric, time-driven )
  • SPIN-PP: ADV REQ
  • SPIN-EC: 高于能量阈值才可以参与
  • SPIN-BC: 广播ADV REQ DATA，收到ADV后等待一段时间再发送REQ，源节点广播DATA，若有其他节点需要DATA，则在自己的timer过期前听到REQ时停止不发
  • SPIN-RL: 若一个节点听到了REQ但是一段时间内未收到DATA，节点认定REQ或DATA丢失，会重新广播REQ，限制源节点发送DATA的频率，发送一个DATA后等待一段时间再发送另一个DATA
Directed Diffusion (data-centric, query-based)
  • sink节点发送interest，通过泛洪让源节点收到

  • 源节点将数据发送至有interest的邻居，一步步直到sink节点

  • 加强信息由sink节点确定(最先、最稳定)发送至邻居节点，然后依次加强直到源节点

  • 沿路节点cache数据，sink节点无需知道整体信息，省电
Roumor
  • 发生事件时发送Agent并维护表，收到Agent的可以更新表
  • 当sink节点进行query时，查看表中有无含interest的event节点数据，若有则建立路由，若没有则随机发送给邻居并探测两条路径是否交会，TTL防止数据包滞留时间过长
GBR
  • 根据节点到sink节点的跳数决定节点的高度，继而决定邻居间的梯度，源节点沿梯度最大的方向发送数据
  • 多邻居梯度相同随机选择
  • 能量低时增加高度，从而减少数据中继量
  • 保持负载均衡

MAC

  • MACA

    • 消灭了隐藏终端问题
    • MACAW添加ACK，当听到RTS但没听到CTS时，也不会听到ACK，有可能是自己听不到，也有可能是根本没发，假设真的没发则要一直等待，故收到CTS时发送者发送DS，若没收到DS证明可以发

  • Z-MAC

    • 负载大时TDMA，负载小时CSMA
    • 开始时使用算法寻找两跳邻居和分配slot，当$F_i$满足$2^{a-1}<=F_i<2^a-1$时TF为$2^a$
    • 每个slot有一个owner，使用此slot传输数据时，owner退避$[0,T_{10}]$，其他node退避$[T_{10},T_{100}]$，若此slot无owner，则使用CSMA来进行分配
    • 当负载低时，任何人均可退避slot的时间进行传输，负载过高时，只有owner一跳范围内可以，使用ECN

  • Y-MAC

    • 帧分成两段，广播和单播期，广播期争夺发送权，单播期每个节点除了在自己的槽上侦听其他都休眠
    • 当第一个receiver接收到帧时，进行channel变换，可以降低延迟并提高吞吐量
    • 缺点是设备需要有多信道

  • L-MAC

    • 分布式算法决定槽的owner，槽用来send
    • 发包的时候分成control message和data unit，control message含到BS的跳数和潜在的receiver，收包时判断自己是不是潜在receiever
    • 从网关开始决定slot-id，直到所有节点选取完，不可选取两跳内存在的，随机选取导致可能选取一样的id，当产生冲突，全网重新选举
    • 对于LMACM，当X离开Y的范围到Z的范围后，若X的id和W的id一致，控制信息传到Z时，Z标记此id为未使用，XW启动选举

  • D-MAC

    • 严格的时钟同步，大幅降低延时
    • 避免冲突使用回退

  • TRAMA

    • 基于流量的slot选举尽量减少给无流量的节点分配slot
    • 降低冲突率，增加休眠时间以及节省能量，缺点是需要存储空间，当某节点在属于自己的slot中且有信息发时才发信息，别的节点发送的对象是自己时接收信息，其他时候休眠
    • random-access期间，节点收或者发拓扑信息，其长度决定节省能量的多少，得到两跳内邻居信息
    • 使用NP算法算出其在某slot的优先级，计算在一个调度周期内优先级最高的slot发送数据并发送调度信息，若无数据发送也通告，本节点的最后一个slot广播下一个调度周期的调度信息

  • LEACH

    • 建立阶段，通过选举算法建立k个簇头，当节点知道自己是簇头后，使用CSMA发送广播，节点选择ADV能量最强的簇头使用CSMA发送Join-REQ帧成簇
    • 稳定阶段，节点在簇头发送的固定slot进行传输，其他时候休眠，簇头一直醒着，收到簇发来的信息并向BS发送
    • 使用直接序列扩频减小簇间的影响，邻居簇间使用不同的扩频序列，保留的扩频序列用于BS传输，簇头向BS传输使用CSMA，传之前探测信道有无同样的扩频码
    • LEACH-C通过各节点把自己的位置和能量信息发送给BS，让BS决定簇头

  • PAMAS

    • 使用两个信道避免数据冲突，一个信道传输数据帧，另一个用来传输控制帧
    • 当sender传输RTS后，若receiver信道无占用则回复CTS，当sender未收到CTS时会使用指数后退算法传输RTS，否则正常传输，两者均发送busy tone，用来阻止邻居干扰

  • S-MAC

    • 一开始需要监听信道一段时间确定规划，若有很多邻居选择和自己一样的规划，延迟$t_d$时间后广播规划减小follower间的冲突，收不到调度信息则自己决定并发给所有的可能邻居
    • 周期性通过SYNC交换调度信息，使用CTS/RTS，若听到CTS/RTS则直接进入睡眠，占空比是之前就确定好了的，可能不大科学

  • T-MAC

    • 通过流量调整休眠和醒着的时段，每个slot的开头侦听一段时间TA，若无消息则继续休眠，当发送，接收或多余听到RTS/CTS时，消息结束后醒一小段时间看有无更多信息
    • 由于休眠太早，可能忽略之后的消息，引起早睡，发送FRTS来提醒下一个节点醒着便于及时发送数据

  • R-MAC

    • 对齐占空参数，一个周期分为三个阶段， SYNC用来同步时钟，DATA阶段用来沿路发送PION(包含sender、下一跳、跳数，使用SIFS)直到到达目的

    • SLEEP阶段用来传输数据，根据PION设定的调度信息来决定唤醒时机

Localization

时间同步
  • 与物理世界时间因果保持一致
  • 有助于减少冗余数据
  • 让WSN的操作和协议的效率更高
影响因素
  • 环境恶劣影响时钟同步
  • 电量过少无法分配更多给时间同步
  • 无线导致效率差以及拓扑变化或节点会突然消失
  • 处理器弱内存少
  • 额外硬件(GPS)
GPS
  • 三颗卫星已经可以定位，但是由于公式$d_i=c\cdot T_i$中，光速是十分快的，故时间同步出现很小的问题时距离的误差会变得极大，卫星中使用铯原子钟，但普通设备显然不可能，故使用4颗卫星减小误差
  • MCS定期收集来自GPS卫星的轨道和时钟信息，并发送回去
  • GPS耗电也贵，不适合用于WS这种小型低能耗设备
